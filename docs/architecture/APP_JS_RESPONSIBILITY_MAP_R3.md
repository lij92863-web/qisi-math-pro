The map is generated from committed source rather than a manually selected subset.
Callers are every lexical `name(...)` occurrence outside the function's own range.
Dependencies combine calls to other inventoried functions and explicit `Qisi.*`
owners. Reactive state is derived from `ref/reactive/computed/shallowRef`
definitions referenced by the function body. Tests are repository test files that
name the production function. A missing lexical caller or test is recorded as
`none detected`; it is not proof of dead code.

Target module and migration wave are planning classifications. They do not mean
the function is already extracted, safe to delete, or production-wired. Each wave
still requires characterization, legacy/new shadow comparison, deterministic
equivalence, production switch, old-owner removal, no-duplicate proof, targeted
tests, true E2E, and full gates.
