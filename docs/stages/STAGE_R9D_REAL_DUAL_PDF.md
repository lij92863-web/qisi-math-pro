# Stage R9D — 双 PDF 内容完整性与真实验收

Date: 2026-07-22

## 范围与边界

- 仅验证 `完整版本题目.pdf` 与 `完整版本答案.pdf`，不扩大到其他付费模型调用。
- 仅允许标准识别链：`qwen-vl-plus`、`qwen-vl-ocr-latest`、`qwen-plus`。
- 不使用“精准”模式，不接受模型猜测的图片归属。
- 答案、解析或图片归属证据不连续时保持 fail-closed；错误绑定比缺失更严重。
- 所有修复基于题号骨架、页面、局部布局和显式结论证据，不包含文件名或固定题号特判。

## 通用修复

- PDF 文本层支持页首拆分题号，例如 `1 1 .`，但仍必须通过完整连续题号骨架验证。
- 题图仅接受本地 PDF 嵌入图片布局证据：题号、页码、图片框、题块框和归属来源必须一致；证据被篡改即拒绝。
- 已验证的本地题图描述符不再被较弱的模型题块框二次否决。
- 空答案只可从同一编号块内唯一的“故选/答案为”锚定结论恢复；冲突或多候选保持未解析。
- 标准模式不再隐式升级到高价精准模型。
- 真实浏览器验收增加题图/解析图角色、精确题号、双图题、刷新持久化、入库、检索、选题、组卷和打印断言。
