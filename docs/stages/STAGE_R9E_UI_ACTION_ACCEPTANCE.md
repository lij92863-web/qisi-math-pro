# Stage R9E — UI 按钮契约与真实交互验收

Date: 2026-07-22

## 边界

- 真实内置浏览器只执行无损状态切换、筛选、导航和本地打印链路。
- 删除、覆盖、批量写入、文件选择和 AI/OCR 等动作不得拿用户现有 IndexedDB 做破坏性试验。
- 所有 117 个点击表达式必须具有唯一 ID、责任模块、风险等级、fixture、副作用和自动或隔离验收方式。
- 页面按钮“有绑定”不是通过标准；自动化动作必须验证可观察状态变化，并同时检查页面错误、控制台错误和 AI/OCR 请求。

## 发现并修复

题目录入页的系统/个人知识点选择器存在真实点击穿透问题：打开一个面板后，覆盖层会拦截另一个选择按钮，导致切换需要点击两次。

修复保持在模板层级边界内：关闭覆盖层位于选择按钮下方，两个选择按钮显式位于覆盖层上方，级联面板仍保持最高层。没有改动知识树、选择或持久化业务逻辑。

新增浏览器行为断言：

- 系统知识点面板可打开。
- 一次点击即可从系统面板切换到个人面板。
- 再次点击个人按钮可关闭面板。
- 题干/答案/解析标签切换后编辑器字段同步变化。
- 题库搜索及五类筛选可变更，重置后全部恢复空值。
- 系统/个人/外部题库模式均可切换并呈 active 状态。
- 三种打印模式均可切换并呈 active 状态。
- 六个顶级视图、批量任务创建/返回均可正常导航。
- 全程 0 个页面错误、0 个 error 级控制台日志、0 次 AI/OCR 请求。
