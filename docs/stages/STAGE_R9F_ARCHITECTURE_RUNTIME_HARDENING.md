## 目标与边界

本阶段只处理已经由代码证据确认的生产卫生、离线启动、诊断输出、安全边界和打印资源生命周期问题。不得改变 DOCX/DOCX、PDF/PDF 的题号所有权、答案/解析对齐、正式入库、IndexedDB 架构或 A4 内容排版策略；不得把冻结研究链路误接入生产，也不得对 `app.js` 做大爆炸式重写。

## 已完成修改

1. 删除 `qisi-batch-importer.js` 中两条各 6,000 多字符的不可达乱码语句，修复活动占位检测正则；新增明确的公式图片占位行为契约。
2. 在 `Qisi.Runtime` 增加统一诊断控制台。普通 `log/info/group/table` 默认静默，仅在 `?qisiDebug=1` 或本机 `qisi:debug=1` 时启用；`warn/error` 永不屏蔽。
3. `app.js`、DOCX pipeline 和 batch importer 接入统一诊断控制台，批处理调试默认关闭。
4. 将 407,279 字节的 Tailwind 浏览器编译器替换为 23,626 字节的预编译静态 CSS，增加可复现的 `npm run build:css`；生产启动不再运行 Tailwind 编译器。
5. 生产入口清单新增四个本地 CSS 的存在性、唯一性和顺序约束。
6. 打印 Blob URL 改由页面生命周期管理，不再固定十分钟回收。
7. 更新存在低危拒绝服务公告的 `body-parser` 间接依赖，`npm audit` 归零。

## 验收标准

- 公式图片占位只能匹配两种显式标记，普通文本和 `displayable` 不得误判。
- batch importer 任意源码行不得再次膨胀到 1,200 字符以上。
- 页面不得加载 Tailwind 浏览器编译器；静态 CSS 必须本地存在并生效。
- 冷启动后页面错误、Tailwind warning、批处理 debug 日志均为 0。
- 恶意 Origin 必须在进入 AI 上游调用前收到 403；无 Origin 的本机程序仍可工作。
- 所有生产 JS 通过语法检查；离线浏览器启动、顶层视图导航和 117 项 UI 动作清单保持通过。
- DOCX/PDF 业务链的完整回归由后续最终材料矩阵统一验收，本阶段不以基础设施改动替代内容真值验证。

## 明确保留的后续风险

- `app.js` 仍是大型协调器。已经完成的 composable 抽取降低了状态密度，但后续只能按独立任务和行为冻结继续缩减，不能为了行数一次性重写。
- 原生 `alert` 30 处、`confirm` 15 处仍存在。它们是体验债务，不是死按钮；替换需要独立的可访问对话框与取消/确认行为矩阵，不能和识别算法混改。
- 两个冻结研究模块仍保留在源码根目录并由生产清单隔离；它们是测试与研究证据，不是运行依赖。
