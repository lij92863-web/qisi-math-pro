## Review Page Improvement Design

### 1. Display Missing Answers Clearly

Current: missing answers shown in problem filter.

Recommended enhancement:
- Show a clear banner: "2 道题答案需人工补充（题号：8, 9）"
- Highlight the specific questions in the draft list
- Differentiate "答案缺失" (no answer at all) from "答案待核对" (answer present but uncertain)

### 2. Show rejectionCode/rejectionDetail

Current: generalized warning in draft.warnings.

Recommended enhancement:
- Display `rejectionDetail` in the review page for rejected answers
- For answer 8/9: show "系统未能自动识别本题答案标签，请手动选择/填入"
- Use `rejectionCode` to categorize the display:
  - `rejection-option-value-not-matched` → "选项值未能匹配"
  - `rejection-multi-option-value-rejected` → "多选题答案未能安全识别"

### 3. Prompt: "解析已识别，答案需人工补"

- For questions where solution exists but answer is missing: show "解析已自动识别，请补充答案"
- Provide a quick-fill UI: dropdown/radio buttons with option labels A-D for single/multi choice
- Allow free text answer for non-objective questions

### 4. Prevent Missing-Answer Submission

- Draft questions with missing answers flagged as "incomplete"
- Submit button disabled or gated with confirmation dialog: "2 道题缺少答案，确认提交？"
- Review summary shows incomplete count before submission

### 5. Mark as Reviewed After Manual Fill

- When teacher fills answer, mark `draft.status` as `reviewed`
- Track `lastReviewedAt` timestamp
- Show reviewed/unreviewed count in batch status
