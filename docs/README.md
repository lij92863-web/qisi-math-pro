# Documentation Index

Documentation status labels:

- **active**: current implementation or operating guidance.
- **superseded**: replaced by a named newer document; retained for traceability.
- **archived**: historical context, not a current instruction.
- **audit-only**: evidence from a completed gate/experiment; do not treat as product behavior.

Primary entry points: [architecture](architecture/README.md),
[testing](testing/README.md), [benchmark](benchmark/README.md),
[release](release/README.md), and [archive](archive/README.md).

`refactor/` and most historical testing reports are audit-only evidence. Files
are not deleted merely because their name contains debug, scan, result, or
recovery terms. The repository cleanup policy and tracked-reference audit must
authorize removal first.
